讓 Agent 動手

讀檔、查資料、操作系統

▶ 看影片 🎓 L2 上手 🎬 7 個段落 🀄 投影片 + 逐字稿
本集開場

歡迎回到《從問 AI 到用 Agent》,這是實作篇的第四集。到這裡,我得老實跟你講:純聊天,已經不夠了。前面幾集我們一直在跟 AI 對話、問它、請它給建議。但你有沒有發現,它講得再好,東西還是在你的電腦裡,它碰不到。這一集,我要讓 Agent 真的動手,去碰你的檔案、你的系統、外面的資料。這集的主角,叫做工具。

1

什麼是「工具」

🔧
  • 1讓 Agent 能對外界動作的能力
  • 2讀寫檔案、上網查資料、執行指令
  • 3沒有工具,它就只是一張嘴
逐字稿

先講清楚,什麼是工具。工具,就是讓 Agent 能夠對外界動手的能力。具體來說,像是讀跟寫你的檔案、上網去查最新的資料、或是在你電腦上執行一段程式、跑一個指令。你可以把它想成是 Agent 的手跟腳。如果只有一個很會講話的腦袋,卻沒有手腳,那它再聰明,也只是一張嘴,講完就結束了。工具,就是把手腳裝上去。

2

有工具跟沒工具,差在哪

⚖️
  • 1沒工具:憑印象回答,可能亂編
  • 2有工具:真的去查、去讀、去做
  • 3結果有沒有根據,差在這裡
逐字稿

那有工具跟沒工具,到底差在哪?差很大。沒有工具的時候,你問它一個問題,它是憑腦袋裡的印象回答你,聽起來很順,但它可能記錯、甚至憑空編一個看起來很像的答案給你。有工具就不一樣了:它會真的去把那個網頁打開來查、真的把那個檔案讀進來看、真的把那件事做一遍。同樣一句回答,一個是憑感覺講的,一個是查證過、做過的,你敢不敢信,差別就在這裡。

3

實際看它動手:讀檔 → 處理 → 存回

🎬
  • 1給它一個真實的檔案
  • 2讓它讀進來、處理
  • 3再把結果存回去
▶ DEMO(Tom 螢幕錄影):給 Agent 一份實際檔案,讓它讀取→處理→輸出
逐字稿

光講你沒感覺,我直接做給你看。接下來我會給它一個真實的檔案,可能是一份原始資料、也可能是一份文件,然後我下一個指令,讓它把這個檔案讀進來、處理一遍,再把結果存回去變成一個新檔。你注意看它的動作:它不是在對話框跟我聊,它是真的去開了我硬碟上的那個檔案。我們來看。

4

回顧:它不是「教你做」,是「做好了」

✅
  • 1它沒有告訴我步驟
  • 2它直接把檔案處理好、存好
  • 3這就是工具帶來的差別
逐字稿

好,看完回來。你有沒有抓到剛剛那個關鍵?它不是告訴我「你應該這樣做、那樣做」,然後把工作丟回給我。它是直接把那個檔案讀進去、處理完、存好,擺在那邊等我驗收。以前它只能出一張嘴,現在它能把事情做完。這一步,就是工具帶來的差別:從「會講」變成「會做」。

5

會做,也代表有風險

⚠️
  • 1它會真的改你的檔、跑你的指令
  • 2改錯了就是真的改下去了
  • 3所以要先設好界線
逐字稿

但是,我要在這裡踩一下煞車。它會做,是好事,可是你要清楚一件事:它是真的在動你的東西。它會真的改你的檔案、真的跑你電腦上的指令。這代表如果它判斷錯了,那個錯也是真的發生,不是在對話框裡講講而已,是真的改下去、真的執行了。能力越大,越要先幫它設好界線。

6

三個保命習慣

🛡️
  • 1先在可回復的環境試,別直接動正式資料
  • 2重要動作,先讓它給你確認再做
  • 3永遠自己驗收,別全盤照收
逐字稿

所以我給你三個保命的習慣。第一,先在一個可以回復的環境試,比如先拿一份備份、一個測試資料夾練手,別一開始就叫它動你唯一一份、動了就回不來的正式資料。第二,重要的、不可逆的動作,像刪檔、覆蓋、送出去,先讓它停下來給你確認,你點頭它再做。第三,也是最重要的,永遠自己驗收,它做完你要親眼看過結果對不對,別因為它很會做就全盤照收。這三條,你越常交辦,越要守。

7

這集帶走一句話

💡
  • 1工具讓 Agent 從「會講」變「會做」
  • 2會做,就代表有風險
  • 3要設界線,也要自己驗收
下篇預告:進階篇 —— 複雜任務、多 Agent、邊界與護欄
逐字稿

如果這一集你只帶走一句話,那就是:工具,讓 Agent 從一個「會講」的嘴,變成一個「會做」的助手。但別忘了,會做的另一面就是有風險,所以你要幫它設好界線,也要自己驗收。到這裡,我們已經讓單一個 Agent 動起來了。可是真正複雜的任務,往往一個 Agent 忙不過來。接下來的進階篇,我們就要進到更大的場面:複雜任務怎麼拆、多個 Agent 怎麼協作、還有最關鍵的,邊界跟護欄要怎麼設。我們進階篇見。
