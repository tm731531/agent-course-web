從問 AI 到用 Agent · 🟢 實作篇

第一個 Agent 上手指南

從零到跑出你的第一個結果

▶ 看影片 🎓 L2 上手 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

大家好,歡迎來到實作篇。前面幾集,我們花了不少時間講觀念:什麼是 Agent、為什麼要從「問」變成「交辦」。聽到這裡,你心裡可能冒出一個很實際的聲音:「道理我都懂了,那我到底要從哪裡開始?」這一集,我就手把手帶你,把你人生第一個 Agent 實際跑起來。我先講結論讓你放心:不用寫程式,也不用花大錢,你今天就可以開始。

1

這集要幫你跨過的那一步

🧭
  • 1觀念都懂了,卻卡在「不知道怎麼開始」
  • 2很多人就停在這裡,再也沒動手
  • 3目標:今天就跑出一個屬於你的結果
逐字稿

我發現很多人學 AI,都卡在同一個地方:觀念聽得很開心,但影片一關掉,人生照舊,因為他不知道第一步該按哪裡。這一集的任務很單純,就是把你從「看懂」推到「動手」。我不會給你一堆理論,我要帶你實際走一遍流程,讓你今天就能跑出一個屬於你自己的結果。哪怕結果很小,只要是你親手交辦出來的,那個手感就完全不一樣了。

2

先把心理門檻放下來

😮‍💨
  • 1你不用很懂技術,不用先學一堆名詞
  • 2不用在電腦裝一堆東西
  • 3從「介面版、訂閱制」開始就好
逐字稿

開始之前,先把你心裡那道牆放下來。我知道一講到 Agent,很多人腦中就浮現黑底白字的程式碼,覺得「這是工程師的東西,我碰不了」。完全不是這樣。你不需要很懂技術,不需要先背一堆專有名詞,也不需要在電腦裡裝一堆看不懂的東西。我們第一步,就從最簡單的「介面版、訂閱制」開始,也就是打開網頁或 App、用對話就能交辦的那種。這個第五集也聊過,對個人來說,這就是最好的起點。

3

第一步:選一個入口

🚪
  • 1個人先用「訂閱制、介面版」的
  • 2判斷原則:能直接對話交辦、操作簡單
  • 3先別糾結挑哪一家 —— 都能練到同一件事
逐字稿

第一步,選一個入口。你可能會問:市面上那麼多家,我到底該選哪一個?我的建議是:先別糾結。對個人來說,你只要抓三個判斷原則:第一,它是訂閱制、有介面的,不用你自己架設備;第二,它能直接用對話跟你交辦,而不是要你寫程式;第三,操作要簡單,打開就能用。符合這三點的,現在主流的幾家其實都可以。你這一集要練的手感,不管用哪一家都練得到,所以別讓「選哪家」變成你不開始的藉口。

4

第二步:開通/註冊

📝
  • 1註冊一個帳號
  • 2選一個「小錢」的方案就好(不用選最貴)
  • 3先不要碰 API、不要碰進階設定
逐字稿

第二步,開通跟註冊。這一步比你想的簡單:註冊一個帳號,然後選一個付費方案。這裡我要提醒兩件事。第一,選那種「一個月一點小錢」的入門方案就夠了,你現在是練手,不需要一開始就上最貴的。第二,你一定會在設定裡看到「API」這種字眼,先跳過它,不要碰。API 是給要寫程式串接的人用的,我們這一集完全用不到。看到看不懂的進階設定,一律先略過,保持簡單。

5

第三步:挑「第一個任務」

🎯
  • 1三原則:最簡單、你一眼能判斷對錯、低風險
  • 2別一上來就丟人生大事給它
  • 3選一件「做壞了也沒差」的小事
逐字稿

第三步,也是最關鍵的一步:挑一個好的「第一個任務」。很多人第一次就失敗,是因為他一開始就丟了一件超難、超重要的事給 Agent,結果做不好就對它失望。挑第一個任務,記住三個原則:第一,要最簡單;第二,你自己要一眼就能判斷它做得對不對;第三,要低風險,就算它做壞了也沒差。這個「低風險」的原則,後面實作篇還會再深談。現在你只要選一件小事,一件就算搞砸了也不會有損失的小事。

6

幾個很好上手的例子

💡
  • 1把一段雜亂的文字整理成條列重點
  • 2幫你比較幾個選項、列出各自優缺點
  • 3幫你草擬一封信、一段訊息
逐字稿

怕你沒靈感,我給你幾個很好上手的例子。第一,把一段雜亂的文字,像是你隨手記的筆記,丟給它,請它整理成清楚的條列重點。第二,你手上有幾個選項在猶豫,請它幫你比較、列出各自的優缺點。第三,請它幫你草擬一封信、或一段要傳給別人的訊息。這幾件事有個共同點:結果好不好,你一眼就看得出來,而且做壞了頂多重來一次,零風險。挑其中一個,就當作你的第一個任務。

7

第四步:練習「交辦」而不是「問」

🗣️
  • 1把「目標」和「你要的結果長怎樣」講清楚
  • 2例:「幫我把這段整理成 5 點重點,每點一句話」
  • 3講完就放手,讓它去做(細節實作篇會深講)
逐字稿

第四步,練習交辦,而不是問。這是整個系列的核心,也是你要開始養的習慣。差別在哪?不要只丟一句「幫我看一下這個」,而是把兩件事講清楚:一,你的目標是什麼;二,你要的結果長什麼樣子。舉例,不要說「幫我整理」,而要說「幫我把這段文字整理成五點重點,每一點用一句話講完」。你把目標跟成品的樣子講清楚,然後就放手,讓它去做。怎麼把話講得更到位,後面實作篇會專門深講,你現在先練「交辦」這個動作就好。

8

第五步:驗收

✅
  • 1看它做出來的,對不對、合不合你要的
  • 2不對 → 補一句話請它調整,再看一次
  • 3「交辦 → 驗收 → 補一句 → 再驗收」就是你之後的循環
逐字稿

第五步,驗收。它做完之後,你的工作是看:這個結果,對不對?是不是我要的?如果不滿意,你不用自己動手改,你只要補一句話,告訴它哪裡不對、想怎麼調,再讓它做一次,然後再看一次。你有沒有發現,這就是一個循環:交辦、驗收、不對就補一句話、再驗收。這個循環,就是你以後每一次用 Agent 的基本節奏。今天你把它跑過一遍,這個節奏就長在你身上了。

9

常見卡關 + 心法

🧗
  • 1第一次一定不完美 —— 這很正常,別追求一步到位
  • 2卡住了就補一句話再試,不是你不行
  • 3重點不是完美,是「先跑出一個結果」
逐字稿

接下來講幾個你一定會遇到的卡關,還有背後的心法。第一,你的第一個結果幾乎一定不完美,這太正常了,連我到現在交辦都常常要來回好幾次,所以千萬別想著一步到位。第二,如果卡住了、它答得牛頭不對馬嘴,那不是你不行,通常只要補一句話講清楚就過了。我要你記住的心法只有一句:這一次的重點不是做到完美,而是先把一個結果跑出來。有了第一個結果,你就從「看影片的人」變成「動過手的人」了。

10

給企業學員的一句話

🏢
  • 1先用「個人帳號」練,不要動公司系統
  • 2拿「不敏感」的東西練手,零風險
  • 3跟公司要不要導入無關 —— 先把手感練出來
逐字稿

這裡特別跟企業的學員講一句。你可能會想:公司還沒決定要不要導入,我現在動這個會不會有問題?不會。因為我要你做的,是用你自己的個人帳號,拿完全不敏感的東西來練手,比如整理一段公開的文字、草擬一封無關緊要的信。這跟公司要不要導入完全是兩回事,是零風險的。個人先把手感練出來,哪天公司真的要推,你已經是那個懂的人了。這一步,穩賺不賠。

11

回顧:五步驟一次看

🔁
  • 1① 選入口 → ② 開通註冊 → ③ 挑第一個任務
  • 2④ 練習交辦(講清楚目標+成品) → ⑤ 驗收
  • 3整條路,不用寫一行程式
逐字稿

我們把今天的路,快速回顧一遍。第一,選一個入口,訂閱制、介面版、能對話交辦。第二,開通註冊,選小錢方案,不碰 API。第三,挑第一個任務,最簡單、看得出對錯、低風險。第四,練習交辦,把目標跟你要的成品講清楚。第五,驗收,不對就補一句話再讓它調。你看,從頭到尾,你完全不用寫任何一行程式。這就是你今天可以立刻走完的一條路。

12

這集帶走一句話

🌟
  • 1別停在「看懂」
  • 2今天就去開通,挑一件最簡單的事,親手交辦一次
  • 3手感,是看影片給不了的
接下來就是實作篇,我們一起動手。
逐字稿

最後,如果這一集你只帶走一句話,那就是:別停在「看懂」。今天,就去開通一個帳號,挑一件最簡單的小事,親手交辦一次。你會發現,原來就這樣,沒有你想像的那麼難。我為什麼一直逼你動手?因為手感這種東西,是看再多影片都給不了你的,你一定要親手跑過一次,它才會變成你的。從下一集開始,我們就正式進入實作篇,一起動手把 Agent 用起來。我們下集見。
