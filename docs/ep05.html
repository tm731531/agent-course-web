從問 AI 到用 Agent · 🟡 觀念篇

花多少、怎麼開始

第 5 集 · 成本邏輯、訂閱 vs API、個人小錢就能試

▶ 看影片 🎓 L1 入門 🎬 13 個段落 🀄 投影片 + 逐字稿
本集開場

歡迎回來。前面幾集,我已經把你說服了,你也大概懂 Agent 到底是什麼、跟以前的問答差在哪。那接下來最現實的問題就來了:這東西到底花多少錢?我一個人,要怎麼開始?這一集我就把這件事講清楚。但在講數字之前,我要先幫你「換一個腦」,因為如果你腦中還裝著舊時代那套算力的想法,你會把整件事的成本想歪。所以我們先從換腦開始。

1

先幫你換一個腦

🧠
  • 1用過 Spark、Databricks 那種人請注意
  • 2你腦中的算力 = 租 GPU 的機器小時數
  • 3這個認知,在 Agent 時代要更新
逐字稿

先講一個很多人會卡住的地方。如果你是上一個時代,用過 Spark、用過 Databricks、跑過大數據的人,那你腦中對「算力」的想像,幾乎一定是這樣:算力等於租一台機器的小時數。你租幾台 GPU、租幾個小時,成本就是這樣算出來的,固定、可預測,租好了就那樣擺著。這個認知本身沒有錯,它在那個時代是對的。但是到了 Agent 時代,這個想法要更新,不然你會拿舊地圖走新的路。

2

舊世界:租 GPU 小時

🖥️
  • 1先開好一批固定產能
  • 2付錢方式 = 時間 × 機器數
  • 3平的、可預測,閒著也在付
逐字稿

我們先把舊世界講清楚。舊世界的邏輯是這樣:你要用算力,你得先開好一批固定的產能,比如我先租五台 GPU 機器擺在那。付錢的方式很單純,就是時間乘上機器數,開幾台、開幾個鐘頭,乘一乘就是帳單。它的特性是「平的」,是一條很平穩、很好預測的線。但它也有一個很痛的地方:就算這幾台機器閒著沒事做,你一樣在付錢,因為你租的是產能,不是你真的用掉的東西。

3

新世界:按 token 計費

📈
  • 1token = 它處理的文字量
  • 2沒有固定產能,隨叫隨用
  • 3花費由「你叫它做多少事」決定
逐字稿

那新世界呢?新世界的邏輯整個換掉了。它不是按時間算,而是按 token 算。token 你先簡單理解成「它處理的文字量」,你丟給它多少字、它回你多少字,加起來就是消耗。這裡最關鍵的差別是:沒有固定產能這回事了,你不用先租、先開一批機器擺著,而是隨叫隨用。你今天用得多就付得多,用得少就付得少,甚至完全不用就是零。你的花費,是由「你到底叫它做了多少事」決定的,不是由時間決定的。

4

尖峰暴衝:漲的是工作量

🚀
  • 1這小時只問一題 → 幾乎是零
  • 2下一小時開 100 個 agent 一起跑
  • 3token 就變 100 倍
逐字稿

這個差別在尖峰的時候會特別明顯,我舉個例子你就有感覺。假設這一個小時,我只丟一個問題進去,那這小時的花費幾乎是零,趨近於沒有。可是下一個小時,我心血來潮,一次啟動一百個 agent 幫我同時跑一百件事,那這一個小時的 token 消耗,可能就直接變成一百倍。你注意一下,這裡漲上去的不是「時間」,時間還是那一個小時;漲的是我派出去的「工作量」。這就是新舊世界最根本的不同:舊世界看你租多久,新世界看你叫它做多少。

5

但要分兩種情況,免得講錯

🔌
  • 1用雲端 API:根本沒在租 GPU,GPU 被抽象掉
  • 2自己養模型:還是 GPU 小時,只是利用率變尖峰
  • 3後者要嘛為爆量預留、要嘛排隊
逐字稿

不過這裡我要很小心地補一句,免得把你帶偏。上面講的按 token,其實還要分兩種情況。第一種,你用的是雲端 API,像是 Claude、像是 GPT。這種情況下,你根本沒有在租 GPU,那些機器對你是完全看不見的、被抽象掉了,你只要按 token 付錢就好,背後有幾台機器不干你的事。第二種,是你自己養模型、自己去買 GPU 機器來跑。這種情況你付的還是 GPU 小時沒錯,但你的機器利用率會從平穩變成尖峰,結果就是:你要嘛為了爆量的時刻先預留一堆機器閒著,要嘛就讓任務排隊等。這兩種是完全不同的成本模型,別搞混。

6

兩種入口:訂閱制 vs API

🎟️
  • 1訂閱(個人):固定月費、額度內隨便用、介面友善、不用寫程式
  • 2API(開發/企業):按 token、沒月費地板、可串進系統、資料流向可控
逐字稿

講完成本邏輯,再來講你實際上要從哪個門走進去。基本上有兩種入口,一種叫訂閱制,一種叫 API。訂閱制是給個人用的:你付一筆固定的月費,在額度裡面隨便你用,而且它有一個很友善的介面,你打開就能用,完全不需要寫任何程式。API 則是給開發者、給企業用的:它是按 token 計費,沒有月費的地板,好處是你可以把它串進自己的系統裡,而且資料流向你可以自己掌控。這兩個入口,對應的是兩種完全不同的人。

7

企業為什麼幾乎只能走 API

🏢
  • 1資料治理與合規的要求
  • 2公司資料不能丟進個人訂閱
  • 3幾乎只能走 API + 自控環境
逐字稿

這裡順便解釋一個常見的疑問:為什麼公司都在講 API,不用訂閱制就好?原因是資料治理跟合規。企業有一堆規定,公司的機密資料、客戶資料,是不能隨便丟進一個個人的訂閱帳號裡的,因為那樣資料流去哪、有沒有被拿去訓練,公司管不到。所以企業幾乎只能走 API,再搭配一個自己能控制的環境,把資料鎖在自己手裡。這是企業的處境。但我要先提醒你下一頁的重點:個人完全不受這個限制。

8

個人不受這個限制

🙋
  • 1企業卡在合規,個人沒這問題
  • 2你不用先想「公司要不要部署」
  • 3先把自己的手感養起來再說
逐字稿

很多人一聽到「要走 API、要自控環境、要顧合規」,就先被嚇退了,覺得這東西門檻好高、好複雜,乾脆不碰。但你要分清楚:那是企業的處境,不是你的。你一個人要開始試用這件事,根本沒有那些合規包袱。你不需要先去想「公司到底要不要部署這個」、「IT 會不會擋」,那些是之後、是別人的問題。你現在唯一要做的,是先把自己的手感養起來,先讓自己知道這東西能幫你做到什麼程度。這兩件事,要完全分開看。

9

個人怎麼便宜起步

🛒
  • 1大約一兩頓飯錢:訂閱月費約 NT$600,更高階約 NT$3000
  • 2實際金額以官網當期為準
  • 3先用訂閱制介面版,別一開始碰 API
金額會變動,請以官方網站當期方案為準
逐字稿

那到底要花多少錢才能開始?講一個你會很有感的比較:大概就是一兩頓飯錢的等級。以目前來說,一般的訂閱月費大約在新台幣六百塊上下,更高階、額度更大的方案大約在三千塊左右。我要特別強調,這個數字會變,實際金額請一定要以官方網站當期的方案為準,不要把我講的數字記死。重點是那個量級:花這樣的小錢,你就能自己先試、先養手感,而且這跟公司未來要不要部署完全是兩回事、零風險。還有一個建議:一開始你就用訂閱制的介面版就好,不要一上來就去碰 API,那個等你有手感了再說。

10

你會得到什麼(一):生活

📚
  • 1行程規劃,幫你排好一整天
  • 2幫你查證、比較(買東西、看合約)
  • 3把一堆雜訊,整理成一個結論
逐字稿

那花這筆小錢,你到底能換到什麼?我分成三塊講,先講第一塊:生活。在生活上,它可以幫你做行程規劃,把一整天、一整趟旅程的安排排好。它可以幫你查證跟比較,比如你要買一個東西,它幫你把幾家的規格、價格攤開來比;你要看一份合約,它幫你把裡面的重點跟眉角挑出來。它還很擅長把一堆亂七八糟的雜訊,幫你整理成一個乾淨的結論。這些都是你每天在做、但很花時間的事。

11

你會得到什麼(二):學習

🎓
  • 1把難懂的資料,變成你的專屬家教
  • 2幫你出考題,測自己懂了沒
  • 3一步一步講解,講到你懂為止
逐字稿

第二塊,學習。這一塊我自己感受特別深。它可以把一份很難懂的資料,直接變成你的專屬家教,你看不懂的地方,就叫它用你聽得懂的方式重講一次。它可以幫你出考題,讓你測測看自己到底是真懂還是假懂。它可以陪你一步一步地講解,你哪裡卡住就停在哪裡,一直講到你真的通了為止。對想學新東西的人來說,這等於身邊隨時有一個不會累、不會不耐煩的老師。

12

你會得到什麼(三):工作

💼
  • 1把重複的、多步驟的工作
  • 2整段交辦掉,不是一步一步問
  • 3你從執行者,變成驗收的人
逐字稿

第三塊,工作。這一塊就回到我們整個系列的核心了。工作上,它最有價值的地方,是可以把那些重複的、要跑很多步驟的工作,整段交辦出去。注意我說的是「整段」,不是你一步一步去問它,而是把一個完整的任務丟給它,讓它自己跑完。這樣一來,你在工作裡的角色就變了:你從那個埋頭把事情做完的執行者,變成一個交辦任務、然後負責驗收的人。這就是前面幾集一直在鋪陳的那個質變。

13

💡
  • 1先換掉「租機器小時」的舊腦
  • 2再花小錢用訂閱制親手試一次
  • 3觀念篇到這裡,前提講完了
下一篇:實作篇,開始動手做給你看
逐字稿

如果這一集你只帶走一句話,那就是這句:先換掉你腦中「租機器小時」那顆舊腦,搞懂新世界是按你叫它做多少事來算的;然後,花一點小錢,用訂閱制,親手去試一次。這兩件事,一個是換腦,一個是動手,缺一不可。觀念篇到這裡就講完了,前面該建立的前提都建立好了。從下一篇開始,我們就進到實作篇,我會真的開始動手,一步一步做給你看。我們下一篇見。
