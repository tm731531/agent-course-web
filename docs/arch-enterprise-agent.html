企業級 Agent 資安治理:地端只是起點

到了企業這一層,問題不再是「省不省成本」,而是「多人怎麼管、資安怎麼守」。地端在這裡,是資安優先的選擇,不是省錢的選擇。

🔑 L4 vs L5 的關鍵差別:L4 地端是「量起來,為省成本/流量搬回家」;L5 企業是「多人 + 合規 + 稽核,地端是為了資安而非省錢」。架構因此完全不同。

① Agent Runtime(執行層)

實際操作任務的 agent 個體,「一人一隻」的個人助理模型。

例:OpenClaw、NemoClaw、各家 copilot

② 治理層(控管層)

集中控管「誰能用哪個 agent、agent 能碰哪些系統」——身分發放、權限授予、稽核留證。

例:Okta、Microsoft Entra Agent ID、CyberArk(現有多為雲端)

③ AI Gateway(過濾層)

攔截 LLM 的輸入/輸出:輸入過濾、輸出控管、配額、PII 遮蔽、稽核。也是目前唯一能地端落地的一層。

例:Kong、Portkey

④ 既有身分基礎(AD)

把企業原本的 Active Directory 帳號體系,和治理層整合起來做 SSO。

⚠️ 現實困境:治理能力與資料落地,目前常是「二選一」——國際治理平台強、但只在雲端;能地端落地的 gateway 只管 LLM 進出、缺完整治理。企業實務上得「三層自組」或找系統整合商客製。
