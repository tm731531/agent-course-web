一次派很多個

多 Agent 並行與分工

▶ 看影片 🎓 L3 進階 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

大家好,歡迎回到《從問 AI 到用 Agent》。前面這麼多集,我們講的都是「一個」Agent 在幫你做事:你交辦一個任務,它自己想辦法完成。但你有沒有想過,有些任務,一個 Agent 做太慢,或者需要好幾種不同的專長?這一集,我要帶你看一個更進階的用法:怎麼「一次派很多個」——也就是多 Agent 的並行與分工。

1

前面都是「一個」在做事

🧍
  • 1你交辦、它執行,一次一個任務
  • 2做得再好,也還是一個人的產能
  • 3有些任務,一個扛不動
逐字稿

我們先回顧一下。到目前為止,你腦中的畫面應該都是:你給一個 Agent 一個目標,它自己去查、去做、去修,最後把成果交給你。這很好用,但它有個天花板——就算這個 Agent 再聰明,它一次也只能專心做一件事,那就是一個人的產能。當任務又大又雜的時候,一個 Agent 就會顯得慢,或者顧此失彼。這時候,我們就需要換一種思路。

2

為什麼要一次派很多個

🎯
  • 1平行加速:同時做很多份
  • 2分工:不同角色做不同子任務
  • 3互相檢查:一個做、一個審
逐字稿

為什麼要一次派很多個 Agent?說穿了有三個理由。第一,是平行加速:一件事拆成很多份,同時開很多個 Agent 一起做,時間就省下來了。第二,是分工:讓不同的 Agent 扮演不同角色,各做各擅長的子任務。第三,是互相檢查:一個負責做,另一個專門挑它的毛病,品質更穩。接下來我一個一個講清楚。

3

理由一:平行加速

⚡
  • 1把一件大事切成很多小份
  • 2很多個 Agent 同時開工
  • 3總時間看最慢那個,不是加起來
逐字稿

先講平行加速。核心概念很簡單:如果一件事可以切成很多互不相干的小份,那你就不必一份一份排隊做,而是同時派出很多個 Agent,每個負責一小份,一起開工。這樣一來,你花的時間不是把每一份加起來,而是取決於最慢的那一個做多久。份數越多、彼此越獨立,省下來的時間就越誇張。

4

平行加速:一個例子

🏭
  • 1要比較 50 家供應商
  • 2一個一個查 → 慢到天荒地老
  • 3派 50 個同時查 → 時間差很多
逐字稿

舉個例子你就有感覺了。假設我要比較 50 家供應商,把每一家的價格、交期、評價都查清楚。如果用一個 Agent,它得一家查完再查下一家,50 家排隊排下去,慢到天荒地老。但如果我一次派 50 個 Agent,一個 Agent 負責一家,大家同時查,最後再把結果彙整起來——原本要等很久的事,可能幾分鐘就有答案。這就是平行的威力。

5

理由二:分工

🧩
  • 1不同 Agent 扮演不同角色
  • 2各做自己擅長的子任務
  • 3像一個小團隊,不是一個全能的人
逐字稿

第二個理由是分工。這跟平行不太一樣:平行是很多人做「同一種」事,分工則是每個人做「不同」的事。你可以讓一個 Agent 專門負責找資料,一個專門負責寫作,一個專門負責挑錯。每個角色只要把自己那一段做好就行。這樣的組合,其實就像一個小團隊,而不是硬要一個全能的人把所有環節都扛下來。

6

分工:一個例子

✍️
  • 1A 負責蒐集與整理資料
  • 2B 根據資料寫成報告
  • 3C 讀一遍、挑毛病、抓錯
逐字稿

舉個寫報告的例子。我可以派三個角色:第一個 Agent 專心去蒐集、整理需要的資料;第二個 Agent 拿著這些資料,把它寫成一份通順的報告;第三個 Agent 則負責從頭讀一遍,挑毛病、抓錯、看有沒有漏掉的地方。三個角色接力,各司其職。你會發現,這跟真實世界一個團隊的分工,幾乎是一模一樣的。

7

理由三:互相檢查

🔍
  • 1讓一個 Agent 專門去挑另一個的毛病
  • 2做的人容易看不到自己的盲點
  • 3有人專門找碴,品質更穩
逐字稿

第三個理由,是互相檢查,這個特別重要。人有個通病:自己做的東西,常常看不到自己的盲點。Agent 也一樣。所以一個很聰明的做法,是讓一個 Agent 負責做,再讓另一個 Agent 專門去挑它的毛病、質疑它、找它的錯。這在專業上叫「對抗式驗證」,講白話就是——安排一個專門找碴的角色。有人盯著,成果的品質就穩多了。

8

別忘了成本(呼應第 5 集)

💸
  • 1一次派 100 個 = 用量也放大 100 倍
  • 2威力很大,帳單也很大
  • 3威力越大,越要主動控管
逐字稿

講到這裡,我得把第五集講成本的那件事拉回來。還記得我說過,有時候一小時 token 會暴衝一百倍嗎?多 Agent 就是那個暴衝的主要來源。你一次派一百個 Agent,等於同時燒一百份的用量,威力放大一百倍,帳單也放大一百倍。這不是要嚇你,而是提醒你:威力越大的工具,越需要你主動去控管它,不能派出去就不管了。

9

什麼時候該用

✅
  • 1任務夠大,值得拆成很多份平行做
  • 2需要好幾種不同專長分工
  • 3重要成果,想要有人互相檢查
逐字稿

那到底什麼時候該用多 Agent?我給你幾個判斷點。第一,任務夠大、而且可以切成很多互不相干的份,平行做能省下明顯的時間。第二,這件事需要好幾種不同的專長,一個角色扛不下來,需要分工。第三,這是個重要的成果,你希望有人互相檢查、把關品質。符合這幾種情況,多 Agent 才真的划算。

10

什麼時候別用

🚫
  • 1小任務,一個 Agent 就綽綽有餘
  • 2步驟環環相扣、拆不開,平行沒意義
  • 3別為了「用」而用,徒增成本
逐字稿

反過來,什麼時候別用?如果只是個小任務,一個 Agent 就綽綽有餘,你硬要拆成好幾個,只是自找麻煩。還有一種情況:任務的每一步都環環相扣,後面要等前面的結果,根本拆不開,那平行也派不上用場。記住,多 Agent 不是拿來炫技的,別為了「用」而用,不然你只是白白多燒成本、多添亂。

11

一個提醒:更難盯

🧭
  • 1同時很多個在跑,你更難全部看住
  • 2目標和驗收標準要說得更清楚
  • 3任務講不清,多派只會更亂
逐字稿

最後給你一個很實際的提醒。一個 Agent 你都要盯著驗收了,一次很多個同時在跑,你當然更難全部看住。所以正因為多,你反而要把目標和驗收標準說得比平常更清楚。這其實就呼應我們一直在講的那句話:任務要交辦清楚。如果你連一個都講不明白,那派十個、派一百個,只會亂上加亂,不會變好。

12

這集帶走一句話

💡
  • 1多 Agent = 把「一個助理」變成「一個團隊」
  • 2但團隊要管理,也要算成本
下集:我實際怎麼用 Agent 處理真實工作。
逐字稿

如果這一集你只帶走一句話,那就是:多 Agent,是把「一個助理」升級成「一個團隊」。團隊能平行、能分工、能互相把關,力量大很多;但團隊也要管理,也要算成本,不是人越多越好。觀念的部分,到這裡就講得差不多了。下一集,我想換個方式,直接讓你看我實際上是怎麼用 Agent,來處理我真實工作裡的事情。我們下集見。
