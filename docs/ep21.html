企業用 Agent 的資安治理

你該想清楚的幾個角度

▶ 看影片 🎓 L5 企業 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

大家好,歡迎回到《從問 AI 到用 Agent》,這是企業篇的第二集。上一集我們講到,企業要用 Agent,問題會從「一個人怎麼用」變成「很多人一起用,而且資安不能出包」。這一集我不打算給你某一套平台、某一張架構圖,叫你照著抄。我要給你的,是一組「想清楚資安治理」的角度。這幾個角度,你以後不管遇到哪一種做法、哪一家的方案,都能拿來套一遍,自己判斷它到底安不安全。這是一套思考框架,不是標準答案。

1

先接上一集

🔁
  • 1上集結論:企業用 Agent = 多人 + 資安
  • 2這集不給你「某個架構」
  • 3給你一組自己就能套的思考角度
逐字稿

先幫還沒看上一集的人接一下。上一集我的結論是:企業用 Agent,跟你自己在家用,最大的差別是兩件事,一個是要開放給很多人用,一個是資安得守得住。那接下來很自然會想問:那到底要怎麼做?這裡我要先講清楚我這集的立場。我不會塞給你一個「照這樣建就對了」的架構,因為每家公司狀況不一樣,抄別人的架構很危險。我要給你的,是幾個「你非想不可」的角度。想清楚這幾個角度,你才有能力去評估任何一種做法。

2

為什麼要用「治理」這個詞

🏛️
  • 1個人:自己對自己負責,錯了重來就好
  • 2企業:很多人、要對公司和法規負責
  • 3所以需要「集中管、可稽核」的機制
逐字稿

我們先搞懂,為什麼企業這邊要特別用「治理」這兩個字。你自己在家用 Agent,本質上是自己對自己負責,它做錯了,大不了你重來一次,沒人會來查你。但企業不一樣:是很多人在用,而且公司要對客戶、對股東、對法規負責。這時候「你自己小心一點」是不夠的,你需要一套「集中在一個地方管、而且事後查得到」的機制,讓每個人怎麼用都在一個規矩底下。這種「集中管、可稽核」的要求,就是治理。它跟個人用最根本的差別,就在這裡。

3

這集的用法:一把可以自檢的尺

🧭
  • 1接下來是五個角度,不是五個產品
  • 2每個角度都是一個「有沒有人想過」的問題
  • 3你拿它去量任何方案,缺哪塊一問就知道
逐字稿

接下來我會講五個角度。我要先提醒你,這五個是「角度」,不是五個要你去買的產品,也不是五層要你去疊的東西。你可以把它們想成一把尺,或者說五個檢查題。以後有人拿一個 Agent 方案來找你,或你自己想在公司推,你就一題一題問下去:這個角度有沒有人想過?有沒有人在守?哪一題答不出來,那就是這個方案的破口。這樣你就不用背架構,也能一眼看出一個做法到底穩不穩。好,五個角度,開始。

4

角度一:身分與權限

🪪
  • 1誰能用哪一隻 Agent、它能碰哪些系統
  • 2不是每個人都該有全部權限
  • 3要能「按人、按角色」發放,也要能收回
逐字稿

第一個角度,身分與權限。核心就一句話:誰,能用哪一隻 Agent,而那隻 Agent 又能碰到哪些系統、哪些資料。這裡最重要的觀念是:不是每個人都該有全部權限。財務在用的 Agent,不該有辦法去翻人事資料;工讀生能碰的範圍,不該跟主管一樣。所以你要問的是:這個做法,能不能按照「人」、按照「角色」去發權限?更關鍵的是,能不能收回?一個人職務換了、或離職了,他名下 Agent 的權限收不收得掉?發得出去、也收得回來,這個角度才算及格。

5

角度二:進出的內容控管

🚦
  • 1進去的:別把敏感資料送出去
  • 2出來的:別冒出不該出現的內容
  • 3像個資這種,要能先遮蔽再處理
逐字稿

第二個角度,進出的內容控管。Agent 背後會接一個很大的語言模型,所以有一個問題你一定要想:哪些東西送進去、哪些東西回出來,有沒有人在把關?往裡送的時候,要防的是敏感資料被無意間送出公司;往外回的時候,要防的是模型講出不該出現的內容。最典型的例子就是個資,也就是常講的 P、I、I,像是客戶的身分證號、電話,這種東西在丟進模型前,最好能先遮蔽掉,用不到就別讓它流出去。所以這個角度的問題是:進出這一關,有沒有一道過濾?還是大家都是原封不動地丟進丟出?

6

角度三:稽核與可追溯

📝
  • 1誰觸發的、讀了什麼、動了什麼,全留紀錄
  • 2出事要查得到,平時要證得出來
  • 3這是企業和個人最大的差別
逐字稿

第三個角度,稽核與可追溯,這個我認為是企業跟個人差最多的地方。稽核要留什麼?三件事:是誰觸發了這個動作、這隻 Agent 讀了哪些資料、它又動了哪些系統。從頭到尾,全部要留下紀錄。為什麼非留不可?因為它有兩個用途:一個是出事的時候,你要查得到,到底是誰、透過哪隻 Agent、動了什麼;另一個是平常沒事的時候,稽核來了,你要能證明給他看,我們是有在管的。你自己在家用,做錯了重來就好;但公司裡答不出「到底誰動了什麼」,這個 Agent 就不敢放進來。所以這一題很硬:全程留不留紀錄?

7

角度四:資料落地與邊界

🏠
  • 1哪些資料「絕對不能出門」,要先劃出來
  • 2地端在這裡是「為資安」,不是為省錢
  • 3呼應 L4 跟 L5 的差別
逐字稿

第四個角度,資料落地與邊界。你要先幫公司劃一條線:哪些資料是絕對不能離開公司的?客戶名單、財報、還沒公開的東西,這些一旦送到外面就收不回來了。劃清楚這條線之後,才會延伸出一個決定:這些不能出門的資料,相關的處理是不是得放在公司自己的機房、自己的環境裡做,也就是所謂的地端。這裡我要特別強調一個觀念,呼應我們前面 L4 跟 L5 講的:企業選擇地端,重點是「為了資安」,為了讓資料不出邊界,而不是為了省那點錢。你把「省錢」跟「守邊界」這兩件事分清楚,這個角度才想得對。

8

角度五:與既有制度整合

🔗
  • 1公司本來就有帳號、權限、稽核制度
  • 2Agent 治理要「接上去」,不是另起爐灶
  • 3少一套要維護、也不會出現管理死角
逐字稿

第五個角度,跟既有制度整合。這個角度很容易被忽略,但超級重要。你要記住一件事:公司不是今天才開始管人的。它本來就有一套員工帳號、有一套權限制度、有一套稽核流程,運作很多年了。所以 Agent 的治理,正確的做法是「接到這套既有的制度上」,而不是另外再造一套全新的、平行的東西。為什麼?因為你另起爐灶,就會多一套系統要維護,而且兩套之間一定會有對不齊的地方,那些縫隙就是管理死角。相反的,接上去,員工用他原本的帳號登入,人一離職、原本的帳號一停,他的 Agent 權限也跟著全斷。這個角度問的是:它接不接得上公司原本的規矩?

9

五個角度,一次複習

🧱
  • 1① 身分權限:誰能用、能碰什麼
  • 2② 內容控管:進出有沒有過濾
  • 3③ 稽核追溯:全程留不留紀錄
  • 4④ 資料邊界:哪些絕對不能出門
  • 5⑤ 制度整合:接不接得上原本的規矩
逐字稿

五個角度講完,我們一次複習,你把這張記起來就好。第一,身分與權限:誰能用、能碰哪些系統,發得出、也收得回。第二,進出的內容控管:送進去、回出來,有沒有一道過濾,個資有沒有先遮。第三,稽核與可追溯:誰觸發、讀了什麼、動了什麼,全程留不留紀錄。第四,資料落地與邊界:哪些資料絕對不能出門,地端是為了守邊界。第五,與既有制度整合:接不接得上公司原本的帳號、權限、稽核。這五題,就是所謂「資安治理」四個字真正的內容。

10

一個務實的心法

🌱
  • 1不用一次做到完美,一步到位反而卡死
  • 2但這五題「有沒有人想過、有沒有人守」
  • 3要先問清楚,再決定要不要放手用
逐字稿

講到這你可能會有壓力:五個角度,每個都要做到滿分,那不就永遠上不了線?我給你一個很務實的心法:不用一次做到完美。真的要求一步到位,通常結果就是卡死、什麼都推不動。重點不是每個角度都滿分,而是這五題「有沒有人想過、有沒有人在守」。你可以先守住最要命的,比如稽核跟資料邊界,其他的邊做邊補。但你要能明明白白地說出:這五個角度,每一個現在是什麼狀態、誰負責。問清楚這個,再決定要不要放手讓大家用。心裡有底地放手,跟閉著眼睛放手,是兩回事。

11

怎麼拿這套角度做決定

🧭
  • 1有人來推一個 Agent 方案 → 五題問一遍
  • 2答得出來 → 心裡有底地放手
  • 3答不出來 → 那就是它的破口,先補再說
逐字稿

那實際怎麼用這套角度?我給你一個很簡單的用法。以後只要有人拿一個 Agent 的做法來找你,不管是外面的方案、還是內部想自己做,你就把這五題依序問一遍:身分權限、內容控管、稽核追溯、資料邊界、制度整合。五題都答得出來、都有人守,那你就可以心裡有底地放手用。只要有哪一題答不出來、或是「還沒人想過」,那個地方就是這個方案的破口,先把它補起來、或者至少寫下來當風險,再決定要不要上。你不需要是資安專家,光是把這五題問出口,你就已經比多數人謹慎了。

12

這集帶走一句話

💡
  • 1企業級 Agent,難的不是讓它更聰明
  • 2難的是「怎麼讓很多人安全地用」
  • 3五個角度守住,才敢在公司真的放手
企業級 Agent = 怎麼讓很多人安全地用
逐字稿

如果這一集你只帶走一句話,就是這句:企業級的 Agent,難的從來不是把它變得更聰明,而是「怎麼讓很多人安全地用」。一隻聰明的 Agent,現在不難找;真正決勝負的,是它外面那一圈——誰能用、進出過不過濾、出事查不查得到、資料出不出門、接不接得上公司原本的規矩。這五個角度守住了,你才敢在公司裡真的放手,讓大家一起用。所以下次在公司想推 Agent,先別急著誇它多聰明,把這五題問一遍,答得出來,再放手。
