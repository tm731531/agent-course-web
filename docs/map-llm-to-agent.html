① LLM 本質

腦只會一個本領:看現有文字,預測下一個字。這是所有 AI 能力的基礎。

② 平台 / 硬體

同一個模型在不同機器上表現天差地遠。要確保「算得對、跑得動、有足夠的上下文空間」。

③ 工具 / Tool Use

腦沒有手,得靠程式代執行工具。它吐出「幫我呼叫 X」的指令,你執行後回報結果。

④ Agent 循環

把工具往返自動化——腦提需求、程式執行、回報結果,重複多輪直到完成任務。

⑤ 選型(5 張成績單)

別只看參數大小,要看多個面向(見下)。

⑥ 微調 / RAG

想讓腦懂公司知識就用 RAG;要它用特定語氣流程就微調。兩者結合最完整。

⑦ 公司資產化

微調後的模型是你的資產,可複製到各廠地端部署,省費用也保隱私。

⑤ 選型的「5 張成績單」

  1. 規模 × 平台 context —— 同模型跨平台差異大
  2. 寫程式(codegen)—— 一次過率與隱形契約遵守度
  3. 工具 / agent —— 多步任務的真實表現
  4. Coder vs reasoning —— 能力傾向的取捨
  5. 格式 / 免費真實性 / 隱私 —— 相容、原廠免費、訓練用途
