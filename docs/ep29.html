A2A 實戰:Agent Card 與溝通方法

怎麼把「Agent 找 Agent」真的架起來

🎓 L3 進階 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

歡迎回到《從問 AI 到用 Agent》。上一集我們講了 A2A,重點是讓一個 Agent 去找「別的 Agent」幫忙。那一集講的是「它是什麼」,這一集我要往上疊一層,講「怎麼把它架起來」。你會發現,真正要動手的時候,核心其實就三件事:那張對外的自我介紹卡怎麼寫、一次協作裡誰是誰、還有它們之間怎麼溝通。我們一件一件來看。

1

這集要架起來的三件事

🧭
  • 1① Agent Card:那張對外的自我介紹卡怎麼寫
  • 2② 角色:一次協作裡誰主動、誰被動
  • 3③ 溝通:兩個 Agent 之間怎麼來回
逐字稿

先給你一張地圖。上一集我們知道了 A2A 是讓 Agent 之間互相交辦,但要真的把它架起來,你只要抓住三件事就好。第一,Agent Card,也就是那張讓別人找得到你的自我介紹卡,這是整集的重點。第二,角色,一次協作裡誰是去交辦的、誰是被交辦的,先分清楚。第三,溝通方法,兩個 Agent 之間到底怎麼來回。這三件事想通了,你就抓住 A2A 的骨架了,細節的協定規格你完全不用背。

2

Agent Card 是什麼(核心)

🪪
  • 1一張放在公開網址上的「自我介紹卡」
  • 2對外講清楚:我是誰、我會哪些能力、怎麼呼叫我、怎麼驗身分
  • 3別的 Agent 讀到卡,就發現你、知道能拿你做什麼
逐字稿

先講最重要的 Agent Card。你就把它想成一張放在公開網址上的自我介紹卡。這張卡對外講清楚幾件事:我是誰、我會哪些能力、你要怎麼呼叫我、還有怎麼驗證我的身分。別的 Agent 只要讀到這張卡,不用事先跟你約定,就能發現你的存在,也知道能拿你來做什麼。這裡我刻意不去背卡片裡有哪些欄位、長什麼格式,那是工具幫你處理的;你要記住的是它的功能——一張讓陌生 Agent 看得懂的名牌。

3

既有系統怎麼「變身」

🔌
  • 1不用把系統重寫成 Agent
  • 2只要幫它掛一張 Agent Card
  • 3它就變成 Agent 世界找得到、用得到的一員
逐字稿

很多人一聽到要加入 Agent 生態,就以為要把手上的系統整個重寫,其實不用。這是 Agent Card 最省力、也最實用的地方:你不必動系統的內部,只要幫它掛上一張 Agent Card,對外宣告它會什麼、怎麼呼叫,它就從一個孤立的服務,搖身變成 Agent 世界裡找得到、也用得到的一員。這是把舊系統接進這個生態最省力的一步——不是打掉重練,而是幫它開一個對外的門面。

4

為什麼「設計好」比什麼都重要

⭐
  • 1卡寫得清楚精準 → 別的 Agent 才用得對、用得到
  • 2卡寫得含糊 → 等於你對外沒開口
  • 3靈魂:把「我會什麼、輸入輸出是什麼」講清楚
逐字稿

這一張我要特別停下來講,因為這是整集的重點中的重點。這張卡設計得好不好,幾乎決定了一切。你把能力寫得清楚、精準,別的 Agent 才用得對、也用得到你;你要是寫得含含糊糊,別人讀了也不知道你到底會什麼、該怎麼交辦,那等於你對外根本沒開口。所以這張卡的靈魂,就是把「我到底會什麼、你給我什麼、我會還你什麼」這件事講清楚。花時間打磨這張卡,比什麼都值得。

5

協定角色:誰主動、誰被動

👥
  • 1發起方:主動去交辦的那個 Agent
  • 2接收方:被交辦、對外開了 Agent Card 的那個
  • 3一次協作先分清楚:誰去交辦、誰被交辦
逐字稿

架起來的第二件事,是分清楚角色。A2A 的一次協作,基本上就兩個角色。一個是發起方,也就是主動跑去交辦、去請人幫忙的那個 Agent。另一個是接收方,也就是被交辦、而且事先對外掛好了 Agent Card 讓人找得到的那個。很簡單,一個主動、一個被動。你在設計的時候,先問自己這一輪裡誰是去交辦的、誰是被交辦的,把這條線分清楚,後面誰要開卡、誰去讀卡,就一目了然了。

6

三種溝通方法

💬
  • 1① 一次性請求-回應:問一件事、拿一個答案
  • 2② 多輪對話:來回澄清、一起把任務做完
  • 3③ 長任務/串流:任務跑很久,邊做邊回報進度
逐字稿

第三件事,是溝通方法。兩個 Agent 對上了,怎麼講話?大致有三種。第一種是一次性的請求跟回應:我問一件事,你給我一個答案,乾脆俐落。第二種是多輪對話:兩邊來回澄清、你一句我一句,一起把一件比較模糊的任務慢慢做完。第三種是長任務,或者叫串流:這件事要跑很久,對方不會讓你乾等,而是邊做邊回報進度。先記住這三個名字,下一張我用白話講什麼情況該用哪一種。

7

什麼情況用哪一種

🔀
  • 1查一個明確答案 → 一次性請求-回應
  • 2任務講不清、要邊做邊對 → 多輪對話
  • 3要跑很久的大工程 → 長任務/串流,別讓人乾等
逐字稿

用白話對一遍你就會挑了。如果只是查一個明確的東西,像問一筆資料、要一個結果,那就用一次性的請求跟回應,一來一回最省事。如果任務一開始講不太清楚,需要邊做邊對、來回確認細節,那就用多輪對話,像兩個人一起討論。如果是那種要跑很久的大工程,一次回不完,那就用長任務串流,讓對方邊做邊回報進度,不會讓你在那邊乾等、也不會不知道它到底跑到哪。挑對溝通方式,協作才順。

8

修車廠例子(上):客戶→店長→技師

🔧
  • 1客戶把車丟給「店長 Agent」交辦(A2A)
  • 2店長找各「技師 Agent」多輪診斷(A2A)
  • 3來回問診:哪裡怪、什麼聲音、什麼時候發生
逐字稿

我用一個修車廠的例子,把角色跟溝通一次串起來。客戶開車進來,把「我的車怪怪的」這件事交辦給店長 Agent,這是一次 A2A 交辦,客戶是發起方,店長是接收方。店長自己不親自修,它轉頭去找底下各個技師 Agent,而且不是問一句就算,是多輪對話地問診:哪裡怪、什麼聲音、什麼時候發生,來回幾輪把問題聊清楚。你看,交辦跟多輪協作,在這一段就都出現了。

9

修車廠例子(下):技師用工具、再找零件商

🛠️
  • 1技師內部用 MCP 呼叫診斷儀、查維修手冊(用工具)
  • 2技師再用 A2A 找「零件供應商 Agent」(再交辦)
  • 3A2A 找隊友、MCP 用工具,分工一看就懂
逐字稿

接著看技師這一端。技師要診斷,它在自己內部用 MCP 去呼叫診斷儀、去查維修手冊——這是「用工具」,是上上一集講的方向。等診斷出要換零件,技師又用 A2A 去找「零件供應商 Agent」下單——這又是一次交辦。整條線走下來你就懂了:A2A 是往旁邊找隊友幫忙,MCP 是往下拿工具補能力,兩個各做各的、合在一起。一個修車流程,把「找 Agent」跟「用工具」的分工講得清清楚楚。

10

MCP + A2A 一起看(收束)

🧩
  • 1架 MCP Server = 開放你的「工具能力」
  • 2寫 A2A Agent Card = 開放你的「Agent 身分」
  • 3不同層,合起來才是完整的對外開口
逐字稿

把這兩件事收束成一句好記的話。你架一個 MCP Server,等於對外開放你的「工具能力」,讓別人的 Agent 拿去用。你寫一張 A2A 的 Agent Card,等於對外開放你的「Agent 身分」,讓別的 Agent 發現你、跟你協作。這是兩個不同的層次:一個開放能力、一個開放身分。你不用二選一,實務上常常兩個都做——把工具掛出去,也把自己的名牌掛出去,合起來,才是一個系統完整的對外開口。

11

呼應脊椎:自動連鎖,越要有邊界

⚠️
  • 1Agent 能自動互相交辦,一連串動作會自動連鎖跑
  • 2你交辦一次,底下可能又交辦好幾層
  • 3威力越大,越要先劃好邊界跟把關
逐字稿

照例呼應我們這系列的脊椎:能力和風險是一體兩面。你看修車廠那條線,客戶只交辦一次,底下就自動連鎖出店長找技師、技師找零件商好幾層。這正是 A2A 厲害的地方,也是要小心的地方:你交辦一次,一連串動作可能就自動接力跑下去,不再每一步都回頭問你。所以能力越大,越要先把邊界跟把關想清楚——誰能交辦給誰、能交辦到多遠,這條線要自己先劃好。

12

這集帶走一句話

💡
  • 1A2A 架起來的核心,就是那張 Agent Card
  • 2把系統寫成一張別的 Agent 讀得懂的自我介紹卡
  • 3卡設計得好不好,決定別人用不用得動你
A2A 系列到此:MCP 用工具、A2A 找隊友、Agent Card 是門面
逐字稿

如果這一集只帶走一句話,那就是這句:把 A2A 架起來的核心,就是那張 Agent Card。你要做的,是把你的系統寫成一張別的 Agent 讀得懂的自我介紹卡——講清楚你是誰、你會什麼、怎麼找你、怎麼跟你協作,它就能加入這個互相交辦的世界。而這張卡設計得好不好,直接決定了別人用不用得動你。工具怎麼接、隊友怎麼找,我們都講過了;把這張卡打磨好,你就真的站上這個生態了。我們下一集見。
