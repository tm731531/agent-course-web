A2A:讓 Agent 找 Agent 幫忙

一群會互相交辦的 Agent,而不是單打獨鬥

▶ 看影片 🎓 L3 進階 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

歡迎回到《從問 AI 到用 Agent》。上一集我們講了 MCP,重點是讓一個 Agent 接上各種「工具」跟資料,幫自己長出能力。這一集要講的是另一個方向:不是接工具,而是讓一個 Agent 去找「別的 Agent」幫忙。這件事有一個名字,叫做 A2A。我們就從它跟 MCP 的差別開始講起。

1

先接住上一集:兩個不同方向

🧭
  • 1上集 MCP:讓 Agent 接「工具」
  • 2這集 A2A:讓 Agent 找「別的 Agent」
  • 3一個往下拿能力,一個往旁邊找幫手
逐字稿

先把這一集擺在上一集旁邊看。MCP 解決的是:讓一個 Agent 去接工具、接資料,替自己補上原本沒有的能力,方向是往下伸手去拿東西。這一集的 A2A 不一樣,它處理的是:當一個 Agent 自己做不完,怎麼往旁邊去找「另一個 Agent」來幫忙。一個是往下拿工具,一個是往旁邊找幫手,是兩件不同的事。

2

本集核心對照:垂直 vs 水平

🔀
  • 1MCP = Agent ↔ 工具/資料(垂直:接工具拿能力)
  • 2A2A = Agent ↔ Agent(水平:交辦給另一個 Agent)
  • 3互補,不是二選一
逐字稿

這是整集最重要的一張對照,請你記住。MCP 連的是「Agent 和工具、資料」之間,你可以想成垂直的:Agent 往下接一個工具,拿到一份能力。A2A 連的是「Agent 和 Agent」之間,是水平的:一個 Agent 把事情交辦給另一個 Agent。這兩個是互補的關係,不是要你二選一;實務上常常是一邊接工具、一邊找隊友,同時在用。

3

它的正式身分

📖
  • 1A2A 全名 Agent-to-Agent(代理對代理)
  • 2Google 在 2025 年提出的開放協定
  • 3和 MCP 定位互補,不衝突
逐字稿

講一下它的正式身分。A2A 的全名是 Agent-to-Agent,直白講就是「代理對代理」。它是 Google 在二〇二五年提出、而且開放出來的一套協定,專門定義「一個 Agent 要怎麼跟另一個 Agent 溝通、交辦任務」。它跟上一集的 MCP 是互補的定位:MCP 管 Agent 怎麼接工具,A2A 管 Agent 之間怎麼合作,兩個各司其職,不會打架。

4

為什麼要讓 Agent 找 Agent

👥
  • 1一個 Agent 不可能什麼都會
  • 2讓專門的 Agent 各做各擅長的
  • 3像一個團隊分工,比一顆硬做全部好
逐字稿

為什麼要有這個方向?因為一個 Agent 不可能什麼都會、什麼都做得好。與其硬要一顆 Agent 從查資料、算數字到寫報告全包,不如讓幾個各有專長的 Agent 分工:擅長查的去查、擅長算的去算,再彼此交辦、把結果串起來。這就像一個團隊,大家各做各擅長的事,整體會比一個人硬扛全部,做得又快又好。

5

用法一:把既有服務包成「能力」

🪪
  • 1給既有服務掛一張「Agent Card」
  • 2常見是一份 agent.json,放在公開網址上
  • 3對外宣告:我是誰、會哪些能力、怎麼呼叫我
逐字稿

A2A 第一種常見的用法,是把一個已經存在的服務,包裝成別的 Agent 讀得懂的能力。做法是:給這個服務掛上一張「Agent Card」,常見的形式就是一份叫 agent.json 的卡片,放在服務的一個公開網址上。這張卡片對外宣告三件事:我是誰、我會哪些能力、還有你要怎麼呼叫我。等於幫服務貼上一張讓人看得懂的名牌。

6

用法一(續):別的 Agent 讀卡就能用

🔎
  • 1別的 Agent 讀到這張卡片,就能發現它
  • 2把它當成一個可用的技能來呼叫
  • 3一個服務,變成 Agent 世界用得到的能力
逐字稿

有了這張 Agent Card 之後會發生什麼?別的 Agent 只要讀到這張卡片,就能「發現」這個服務的存在,知道它會什麼、怎麼呼叫,然後把它當成自己手上一個現成的技能來用。這麼一來,原本一個孤立的服務,就搖身變成「Agent 世界看得懂、也用得到」的一個能力,別的 Agent 需要的時候,交辦給它就好。

7

用法二:Agent 真正的對外接口

💬
  • 1A2A 也可以是一個 Agent 對外開的門
  • 2讓另一個 Agent 直接對話、交辦、來回協作
  • 3不只是呼叫一個固定功能
逐字稿

第二種用法更進一步。A2A 也可以是一個「真正的 Agent」對外開的一道門:讓另一個 Agent 直接跟它對話、把任務交辦給它、甚至來回討論、多輪協作,而不只是呼叫一個寫死的固定功能。這時候兩邊都是會思考、會動手的 Agent,是真的在合作把一件事情完成,而不是一邊單純去按另一邊的按鈕。

8

把兩種用法收成一句

🧵
  • 1一種:讓既有服務被 Agent 讀懂、被呼叫
  • 2一種:讓 Agent 之間真的互相交辦協作
  • 3同一個協定,兩種深度
逐字稿

把這兩種用法收攏成一句好記的話。第一種,是讓一個既有的服務被 Agent 讀懂、被當成技能呼叫,偏向「把東西掛上來讓人用」。第二種,是讓兩個 Agent 之間真的互相交辦、來回協作,偏向「真正的合作」。它們用的是同一套 A2A 協定,只是深淺不同:一個是被動被呼叫,一個是主動一起做事。

9

Agent Card 為什麼是關鍵

🪧
  • 1就是一張「自我介紹卡」
  • 2陌生的 Agent 不用事先約定也能發現彼此
  • 3這是「Agent 互相找得到」的基礎
逐字稿

這裡單獨把 Agent Card 拉出來講,因為它是整件事的地基。你就把它想成一張「自我介紹卡」:上面寫著我是誰、我會什麼、怎麼找我。有了這張卡,兩個原本互不相識、事先也沒約定過的 Agent,也能發現彼此、知道對方會哪些能力。這就是「Agent 之間互相找得到」最基礎的一塊,沒有它,大家就只能各做各的。

10

呼應脊椎:能力越大,越要有邊界

⚠️
  • 1Agent 能自動找 Agent、自動交辦,威力更大
  • 2但一連串動作可能自動連鎖跑下去
  • 3更要有邊界跟把關(呼應風險那集)
逐字稿

這裡要呼應我們這系列的脊椎:能力和風險是一體兩面。當 Agent 不只會用工具,還能自動去找別的 Agent、自動把任務交辦出去,威力當然更大;但反過來想,這也代表一連串動作可能一個接一個、自動連鎖地跑下去,不再每一步都回來問你。所以能力越大,越需要清楚的邊界跟把關,這正是我們風險那一集一直強調的事。

11

企業要用,更要管誰能跟誰講話

🏢
  • 1管好:誰能跟誰對話
  • 2管好:能交辦什麼、不能交辦什麼
  • 3自動連鎖越方便,控管越不能省
逐字稿

放到公司的場景,這件事更要謹慎。當一群 Agent 可以互相交辦,你就得管清楚兩件事:第一,誰能跟誰講話,不是任何 Agent 都能隨便呼叫另一個;第二,能交辦什麼、不能交辦什麼,得先劃好線。自動連鎖越方便,越不能省掉這層控管,不然一個交辦接一個交辦,跑到哪裡你自己都不知道。這部分下一集會專門講。

12

這集帶走一句話

💡
  • 1MCP 讓 Agent 拿到工具
  • 2A2A 讓 Agent 找到隊友
  • 3未來的 Agent 不是單打獨鬥,是一群會互相交辦的 Agent
下集:企業怎麼控管這些(MCP / 技能 / Agent 互連)
逐字稿

如果這一集只帶走一句話,那就是這句:MCP 讓 Agent 拿到工具,A2A 讓 Agent 找到隊友。你完全不用去懂協定的細節,但你要記得一件事——未來的 Agent 不會是一顆孤零零單打獨鬥的腦袋,而是一群會互相發現、互相交辦、互相協作的 Agent。它們越能自動合作,我們就越需要學會控管;下一集,我們就正面來談企業怎麼管好這一切。我們下一集見。
