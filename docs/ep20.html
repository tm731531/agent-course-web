企業用 Agent,為什麼是完全不同的問題

企業篇(L5)開場 · 從省成本到守資安

▶ 看影片 🎓 L5 企業 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

大家好,歡迎來到企業篇的第一集。前面我們一路從個人怎麼用 Agent 講到地端,那時候把 AI 搬回自己家,講白了是「量起來了、為了省成本」。可是這一集我要告訴你,一旦進到企業,問題整個換了一個。它不再是省不省錢,而是「這麼多人一起用,到底怎麼管、資安怎麼守」。這一個層級,我把它叫做企業級,今天先帶你看清楚它為什麼跟前面完全不一樣。

1

先回到個人:一顆腦,自己用

🧍
  • 1一個人、一顆 Agent、自己交辦
  • 2出事自己扛、資料自己看
  • 3沒有『別人』要管
逐字稿

我們先回到個人的情境。個人用 Agent,說到底就是一個人、一顆腦、自己用。我交辦、我驗收,做壞了我自己扛,資料就在我自己電腦裡,我自己看。整個過程裡沒有「別人」這件事,所以我根本不需要去想什麼權限、什麼稽核,反正就我一個人,愛怎麼用怎麼用。這是個人級最舒服的地方,也正好是它跟企業最大的分水嶺。

2

企業:幾百人一起用很多 Agent

🏢
  • 1不是一個人,是幾百人、各部門
  • 2各有各的權限、各碰各的系統
  • 3同時跑很多不同的 Agent
逐字稿

那企業呢?企業不是一個人,是幾百人,分成一堆部門,每個人的權限還不一樣,能碰的系統也全都不同。然後這幾百人,同時在用很多顆不同的 Agent。你把個人那顆自己用的腦乘上幾百倍,再讓它們去碰公司各種系統,你就會發現,這已經不是同一個問題,而是一個關於「一群人」的問題。

3

新問題一:誰能用、能碰什麼(權限)

🚪
  • 1誰可以用哪一顆 Agent
  • 2那顆 Agent 又能碰哪些系統
  • 3個人根本不用想這件事
逐字稿

人一多,個人時代從來不用想的問題就冒出來了。第一個是權限:誰可以用哪一顆 Agent?那顆 Agent 又能碰到公司哪些系統、哪些資料?這在個人時代不存在,因為就我一個人,我自己就是全部權限。但在企業裡,你總不能讓每個人的 Agent 都打開所有系統。這條「誰能用、能碰什麼」的線,必須有人畫、有人管。

4

新問題二:出事誰負責、查得到嗎(稽核)

🔍
  • 1Agent 動了什麼、什麼時候動的
  • 2出了包,要追得回是哪一次、哪個人
  • 3查不到 = 沒人敢負責
逐字稿

第二個新問題是稽核。Agent 會自己動手,那它到底動了什麼、什麼時候動的、是誰交辦的?萬一出了包,你要能一路追回來,是哪一次任務、哪一個人、碰了哪個系統。如果查不到,那就等於沒人敢為它負責。個人時代你不用記帳,因為就你自己;可是企業要能對每一個動作交代得清清楚楚,這叫稽核,是企業級跑不掉的一關。

5

新問題三:資料會不會外流(資安)

🛡️
  • 1餵給 Agent 的公司資料會不會出門
  • 2它產出的東西會不會夾帶機密
  • 3一個人漏,是整間公司的事
逐字稿

第三個問題是資安。你餵給 Agent 的,是公司的客戶資料、合約、財報,這些東西會不會就這樣被送出門?它產出的內容,會不會不小心夾帶了機密?個人漏一點頂多是自己的事,但在企業裡,任何一個人漏,那是整間公司的事,可能是法規、是客戶信任。所以「資料跟輸出會不會外流」,在企業級是被放到最前面盯的一件事。

6

為什麼舊思維一定會出事

⚠️
  • 1把個人的『隨便交辦』直接搬進公司
  • 2等於人人都能讓 Agent 亂碰、亂送
  • 3沒人管得住,遲早爆
逐字稿

這裡有個很多人會踩的坑:把個人那套「隨便交辦、爽就好」直接搬進公司。後果是什麼?等於每一個人都能讓自己的 Agent 隨便碰系統、隨便把資料送出去,沒有一條線攔著。平常沒事,但只要出一次包,你連是誰、動了什麼都查不到。個人的自由,搬到企業就變成失控。舊思維不是不好,是它天生就沒有「管一群人」這一塊。

7

關鍵轉變:地端不再是為了省錢

🔄
  • 1L4 搬回地端:為了省成本
  • 2企業搬回地端:為了資安
  • 3資料不能出門、要能證明給稽核看
逐字稿

接著是最關鍵的轉變。前面 L4 把 AI 搬回地端,動機是省成本,量大了自己養比較划算。可是到了企業,搬回地端的理由整個換掉,變成「為了資安」。資料不能出門,要留在公司自己的機房;而且你得證明給稽核、給法規看,證明它真的沒出去過。同樣是地端,個人是為了錢,企業是為了守得住,動機完全不同。

8

所以架構會長得不一樣

🧩
  • 1要『集中控管』——一個地方管權限、看紀錄
  • 2又要『資料落地』——東西留在自己機房
  • 3兩件事得同時成立
逐字稿

動機變了,架構自然也得跟著變。企業要的比較刁鑽:一方面要「集中控管」,有一個地方統一管誰能用、統一留下所有紀錄;另一方面又要「資料落地」,東西全留在自己機房。集中管、又要留在家裡,兩件事得同時成立。這比個人那顆自己用的腦複雜多了,是個更大的題目,下一集我會專門把它拆開來看。

9

心裡先有個譜:會分成幾層

🧱
  • 1跑任務的那一層(Agent 本身)
  • 2管權限、記紀錄的那一層(治理)
  • 3看內容、擋外流的那一層(閘道)
逐字稿

你現在心裡有個譜就好,不用背。企業要把這件事管起來,大概分成幾層:最底下是真正在跑任務的 Agent 本身;中間一層專門管「誰能用、能碰什麼」,還把所有動作記下來,這叫治理;最上面一層,專門看每一次進出的內容、把該擋的擋下來,常被叫做閘道。分層,就是企業級跟個人級最明顯的差別。

10

這些角色,都有成熟做法

🔌
  • 1管身份、管進出,都不是 AI 才有的新問題
  • 2各層在企業資安早有成套解法
  • 3你不用從零造,先懂每層在解什麼
逐字稿

講到這你可能會怕,這麼多層是不是要自己從零蓋?其實不用。這些角色——管身份、管誰能登入、管流量進出、幫你擋跟記——都不是 AI 時代才冒出來的新問題,企業資安領域本來就有一整套成熟的做法在解。我這裡不細講是哪些工具,重點是讓你知道:每一層要解的問題都真實存在,而且都有人在解。你要做的,是先看懂每一層各守哪一關,之後再去挑合適的方案。

11

一個誠實的提醒

🙏
  • 1不是每個人都得碰到這一級
  • 2但你想在公司推 Agent,遲早要面對
  • 3先知道有這道牆,不會被嚇到
逐字稿

這邊給你一個誠實的提醒。企業級這一關,不是每個人都非碰不可。你如果就自己一個人爽用,前面幾集就夠你走很遠了。但只要你動念,想在公司把 Agent 推起來、讓同事一起用,遲早會撞上今天這三道問題:權限、稽核、資安。我先把這道牆指給你看,不是要嚇你,是讓你到那天不會慌,知道它本來就在。

12

這集帶走一句話

💡
  • 1個人用 Agent:拚的是效率
  • 2企業用 Agent:拚的是『讓很多人安全地用』
  • 3問題的重心,整個換了
下集:資安治理要想清楚的幾個角度。
逐字稿

如果這一集你只帶走一句話,就是這句:個人用 Agent,拚的是效率,怎麼把自己的事做快做好;企業用 Agent,拚的是另一件事,是「怎麼讓很多人一起、安全地用」。從效率變成安全,問題的重心整個換了位置,這就是企業為什麼是全新的題目。下一集,我給你一組「想清楚資安治理」的角度,讓你套任何方案都能自己判斷安不安全。我們下一集見。
