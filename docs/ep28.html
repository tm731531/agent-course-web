架一台 MCP Server

把你自己的能力,開放給 Agent 用

▶ 看影片 🎓 L3 進階 🎬 12 個段落 🀄 投影片 + 逐字稿
本集開場

歡迎回到《從問 AI 到用 Agent》。上一集講 MCP 的時候,我把它比喻成 AI 世界的通用接頭,讓 Agent 可以用一種標準的方式接上各種工具。那一集,你是那個「拿別人做好的 server 來插」的人。這一集,我們往前再走一步:換你當那個「架 server」的人,把你自己、或你公司的能力,做成任何 Agent 都能接上來用的東西。

1

先接回上一集的定義

🔄
  • 1MCP = Agent 對外或對內操作能力的標準
  • 2server 就是「被操作的那一端」
  • 3上集你當使用者,這集你當提供者
逐字稿

先把上一集的定義接回來。MCP 是一套標準,規定 Agent 要怎麼去操作外面或裡面的各種能力。而所謂的 server,講白了就是「被操作的那一端」。你可以想成:Agent 是伸手要用東西的人,server 是那個把東西擺出來、等著被拿的人。上一集我們站在 Agent 這邊當使用者,這一集,我們要換到 server 這邊,當提供能力的人。

2

MCP Server 到底是什麼角色

🧰
  • 1它站在「工具那一端」
  • 2把一個能力包成標準可呼叫的 server
  • 3查資料、呼叫一個服務、讀一份檔…都算
逐字稿

那 MCP server 到底扮演什麼角色?它站在整個流程裡「工具的那一端」。它的工作,是把某一個能力,包裝成一台 Agent 能用標準方式呼叫的 server。這個能力可以很單純,比如幫忙查一筆資料、幫忙呼叫某個服務、幫忙讀一份檔案。重點不是能力多厲害,而是:它被包成了一個標準的門面,誰都能照同一套規矩來敲門。

3

一台 Server 對外宣告三種東西

📋
  • 1可呼叫的動作:tools
  • 2可讀的資料:resources
  • 3可用的提示:prompts
逐字稿

一台 MCP server 架好之後,會對外宣告三種東西,你先記住這三類就好。第一種是可以呼叫的動作,叫 tools,就是「你可以叫我做這些事」。第二種是可以讀取的資料,叫 resources,就是「你可以來我這邊讀這些內容」。第三種是現成可用的提示範本,叫 prompts。Agent 當使用方連上來,就是照標準協定,去看你宣告了哪些、然後挑著用。

4

架設一台 Server:四個步驟

🗺️
  • 1① 決定開放哪些能力
  • 2② 把每個能力定義成一個工具
  • 3③ 跑成一台 server
  • 4④ Agent 端設定連上
逐字稿

接下來講怎麼架。整件事拆成四個步驟,我先給你一張全景,後面再一個一個展開。第一步,決定你要開放哪些能力。第二步,把每一個能力,定義成一個清清楚楚的工具。第三步,把這些工具跑成一台實際在運作的 server。第四步,到 Agent 那一端做設定,讓它連上你這台 server。四步走完,一台能被 Agent 使用的 server 就成形了。

5

步驟一:決定開放哪些能力

🎯
  • 1先想清楚「要讓 Agent 做什麼」
  • 2只挑真的需要開放的
  • 3不是把手邊全部能力一次攤開
逐字稿

第一步,決定開放哪些能力。這一步其實是在做取捨,不是把你手邊會的東西全部攤出來。你要先想清楚:我到底想讓 Agent 幫我做哪些事?然後從裡面,挑出真的值得、也適合開放的那幾項。開得太少,Agent 幫不上忙;開得太雜,後面又難管。所以這一步花點時間想清楚範圍,是整台 server 好不好用、好不好顧的關鍵。

6

步驟二:把能力定義成一個工具

🏷️
  • 1取一個一看就懂的名字
  • 2講清楚:要餵它什麼(輸入)
  • 3講清楚:它會給什麼(輸出)
逐字稿

第二步,把每一個能力定義成一個工具。這裡的重點是「講清楚」,因為 Agent 是靠你的描述來判斷該不該用、怎麼用。每個工具要做三件事:取一個一看就懂的名字;講清楚要餵它什麼,也就是輸入是什麼;再講清楚它會回你什麼,也就是輸出是什麼。名字和說明寫得越清楚,Agent 就越不會用錯,這一步偷懶,後面就會出亂子。

7

步驟三、四:跑起來 + 接上去

🚀
  • 1③ 把這些工具跑成一台 server
  • 2④ 到 Agent 端設定,連上這台 server
  • 3兩邊講同一套協定,就通了
逐字稿

第三步和第四步,把它接起來。第三步,把你定義好的那些工具,跑成一台實際在運作的 server,讓它待在那邊等人來呼叫。第四步,回到 Agent 那一端,做個設定,告訴 Agent「有這麼一台 server,連上去」。因為兩邊講的是 MCP 這同一套協定,設定好之後就通了,Agent 就看得到你宣告的那些工具,可以開始用了。

8

資料怎麼流:走一次完整來回

🔁
  • 1Agent 先問:你有哪些工具?
  • 2選一個、帶上參數,呼叫
  • 3server 執行 → 回結果 → Agent 接著做
逐字稿

把一次完整的來回走一遍,你就懂資料怎麼流了。Agent 想做一件事,它不會亂猜,而是先問 server:你有哪些工具?server 把宣告的清單回給它。Agent 從裡面挑一個合適的,帶上該帶的參數,發出呼叫。server 這邊實際去把事情執行掉,再把結果回傳給 Agent。Agent 拿到結果,才接著做下一步。一問、一選、一呼叫、一回傳,就這樣一圈。

9

對外 vs 對內:同一套做法

🔀
  • 1對內:接公司內部的系統
  • 2對外:接外部的服務
  • 3差別只在:開放給誰、放在哪
逐字稿

這裡呼應一下開頭的定義。同樣一套「架 server」的做法,你可以拿來接公司內部的系統,這叫對內;也可以拿來接外面的服務,這叫對外。做法是一樣的,差別只在兩件事:你這台 server 是開放給誰用、以及你把它放在哪裡。想通這點你就會發現,對內和對外並不是兩種技術,而是同一種技術,擺在不同的邊界上而已。

10

為什麼值得自己架一台

💎
  • 1把你的獨門能力變成標準能力
  • 2任何支援 MCP 的 Agent 都能接
  • 3一次架好,到處都能用
逐字稿

那為什麼值得花力氣自己架一台?因為它會把你、或你公司的獨門能力,變成一種標準能力。原本這個能力只有你會、只有你能操作;做成 MCP server 之後,任何支援 MCP 的 Agent,都能用同一套方式接上來用。你等於是一次架好,到處都能接,不用為每一個 Agent 各寫一套接法。你的能力,就這樣被放大到整個 Agent 世界。

11

一個務實提醒

⚠️
  • 1架 server = 對外開放能力
  • 2架的時候就要想:開放什麼、給多少權限
  • 3別把不該給的也開出去
逐字稿

最後給一個務實的提醒。架一台 server,本質上就是在對外開放能力。所以你在架的當下,就要同時想清楚兩件事:這台到底要開放什麼、以及每一項給到多少權限。方便的另一面,是開得太寬,Agent 能碰的就超出你原本的預期。這不是要嚇你,而是提醒你:開放和邊界,是同一個動作的兩面,別圖方便把不該給的也一起開出去。

12

💡

  • 2讓整個 Agent 世界都用得到
  • 3你不用馬上會寫,但要知道有這條路
