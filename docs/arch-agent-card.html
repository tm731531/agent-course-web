Agent Card:讓既有系統变身成 Agent

你不用把系統重寫成 Agent。只要幫它掛一張「Agent Card」——一份放在公開網址上的自我介紹卡——它就變成 Agent 世界找得到、用得到的一員。這張卡設計得好不好,決定別人用不用得動你。

🪪 三方關係:發起 Agent ↔ Agent Card ↔ 你的系統

重點:別的 Agent 只透過「卡」認識你,碰不到你系統的內部細節。卡上寫什麼、開放哪些 skill,就決定了它能請你做什麼——所以卡本身就是你對外的邊界。

📇 一張好的 Agent Card 該講清楚四件事

卡上宣告白話寫不清楚會怎樣
我是誰這個 Agent 的身分、用途別人不知道你是幹嘛的,不會來找你
我會哪些 skill能被交辦的能力清單能力含糊 → 別人不知道能請你做什麼
怎麼呼叫我入口位置、要帶什麼、回什麼接不上 → 等於沒對外開口
怎麼驗證身分誰有權呼叫、怎麼認證沒把關 → 誰都能來用你的系統

🔐 授權設計:別人憑什麼能用你

Agent Card 不是只宣告「我會什麼」,更要宣告「誰、憑什麼、能用哪些」。授權沒設計好,等於把系統對全世界打開。四層把關:

1卡上宣告驗證方式怎麼證明你是你
卡片講清楚要用哪種身分憑證(例如金鑰、權杖),呼叫方才知道該帶什麼來。沒宣告 = 誰都能來。
2呼叫時附上憑證出示身分
別的 Agent 依卡片指示,呼叫時附上自己的身分憑證,而不是匿名闖入。
3系統驗身分 + 查權限是你嗎?你能用嗎?
系統先確認「是你本人」(驗證),再確認「你被允許用這個 skill」(授權)。兩關都過才放行。
4最小授權只給該給的
每個呼叫方只開放它真正需要的 skill 與資料範圍,不是一把全開。範圍越小,出事波及越小。

🎯 為什麼「卡設計得好」比什麼都重要

1卡是唯一的對外介面別人只看卡
別的 Agent 不讀你的程式碼、不進你的資料庫,它只讀這張卡。卡講得多清楚,別人就能用得多對。
2变身,不用重寫最省力的一步
既有系統原封不動,掛一張卡就加入 Agent 生態。這是把舊系統接進 AI 世界最省力的路。
3卡=邊界先想清楚開放什麼
卡上開放哪些 skill、要不要驗證,就是你對外的權限邊界。開放前先想清楚,別把不該給的也寫上去。
